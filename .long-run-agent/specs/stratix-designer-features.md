# Stratix Designer 英雄设计器 - Feature 拆分文档

## 模块概述
Stratix Hero Designer 是星策系统的 Agent 可视化配置模块，负责 Agent 的创建、编辑、模板管理和配置标准化转换。

## 模块路径
`src/stratix-designer/`

## Feature 清单

### Feature 005: 核心类 StratixHeroDesigner
**优先级**: P0 (最高)  
**预估时间**: 1 天  
**状态**: 待开发

#### 功能描述
实现英雄设计器核心逻辑类，包括：
- 单例管理
- 创建新英雄（基于模板）
- 保存英雄配置
- 导入/导出英雄配置
- 获取预设模板

#### 依赖
- stratix-core (stratix-protocol.ts)
- stratix-core (StratixEventBus)
- axios
- file-saver

## 开发顺序
1. **Feature 006** (配置校验与转换工具) - 基础工具
2. **Feature 007** (预设模板库) - 提供模板数据
3. **Feature 005** (核心类) - 核心逻辑
4. **Feature 008-011** (编辑器组件) - 可并行开发
5. **Feature 012** (主表单组件) - 整合所有组件

## 依赖关系
```
stratix-core (数据协议 + 事件总线)
    └── Feature 006 (校验工具)
            └── Feature 007 (模板库)
                    └── Feature 005 (核心类)
                            └── Feature 008-011 (编辑器组件)
                                    └── Feature 012 (主表单)
```

## 总预估时间
**6 天**

## 技术栈
- TypeScript (主要语言)
- Vue 3 (前端框架)
- Element Plus (UI 组件)

## 注意事项
1. 所有 Agent ID 必须以 `stratix-` 开头
2. 配置校验必须包含所有必填字段检查
3. 导出的配置必须兼容 OpenClaw 原生格式
4. 不处理配置下发，仅输出标准化配置
5. 通过事件总线同步配置更新
